開挖擋土支撐計算工具 本機服務工具

FastAPI + React + SQLite｜支撐構件檢核與報表輸出

工具狀態

目前定位 已納入工具箱索引，但不是純靜態 HTML；需啟動本機後端服務後使用。
使用入口 此頁是 launcher 與邊界說明；實際操作介面由 FastAPI 掛載前端，預設網址為 http://127.0.0.1:8000。
主要流程 匯入 .RIO / .LST / .o 分析文字檔，整理支撐、橫擋、斜撐、大角撐與柱構件檢核，並輸出報表。
資料邊界 SQLite、專案 JSON、匯入檔與報表輸出存於 app_data/；該資料夾屬本機工作資料，不進版本控管。
目前限制 柱構件檢核仍屬第一版移植；複雜案例應與既有 Excel / 計算書交叉比對。

啟動方式

啟動本機服務
cd C:\Users\USER\Desktop\AI\小工具製作\開挖擋土支撐 .\start_html_mode.ps1 -OpenBrowser
停止本機服務
cd C:\Users\USER\Desktop\AI\小工具製作\開挖擋土支撐 .\stop_html_mode.ps1
健康檢查
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8000/api/health
開啟本機服務 檢查 API health 查看 README 返回結構工具箱

服務邊界

本機服務，不是靜態頁 需要 Python FastAPI 後端；若前端建置失敗，腳本會退回 Vite dev proxy 模式。
工程資料留在本機 分析檔、SQLite、PDF/DOCX 報表與專案狀態均屬本機資料，預設排除於 repo 外。
不可直接當靜態部署 若要公開部署，需另行規劃 API、資料儲存、權限與前端 base path。
若要正式納入靜態部署，需另外處理前端 base path、API 服務部署與資料檔儲存邊界；目前此入口以本機工程工作流為準。
