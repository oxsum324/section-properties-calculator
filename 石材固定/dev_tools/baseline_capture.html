🛡️ Baseline Capture 工具 – 重構護欄

本頁是「金樣本（Golden Samples）擷取流程」的入口。在 calc-core 抽取重構前， 先用此工具凍結現況計算結果，作為 refactor 的不變條件。

為什麼需要這個？
若直接 refactor 公式核心而沒有事先凍結現況，重構完成後只能驗「新系統自洽」， 無法保證「沒有意外改變現有計算結果」。Characterization Testing 是工程治理的標準護欄。

📋 操作流程

  1. 選一個尚未拍攝的代表性工程案例（建議從下方「目標案例清單」選一組）
  2. 點下方「啟動 V2 開發模式」按鈕，會以 ?dev=1 開啟 V2
  3. 在 V2 中輸入該案例完整參數，按「🔄 重新計算 / 預覽」確認結果
  4. 右上角浮現的 🛡️ Dev Capture 浮層內，選 case_purpose、填 case_id 與 notes
  5. 勾「自動去識別化」（提交至 git 必勾）
  6. 點「📸 Capture Baseline」，瀏覽器自動下載 case_xx_*.json
  7. 檢查下載檔，確認沒洩漏業主資訊；放入 tests/golden/
  8. 在 tests/golden/_approval_log.md 新增一筆 entry

🔒 治理規則摘要



    

  • 不可逆：baseline 一旦提交，後續變更必須走 baseline:approve --reason="…" 流程
