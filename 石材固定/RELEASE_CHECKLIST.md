# 石材計算書正式輸出檢查清單

此清單用於正式送審、交付業主或歸檔前。程式問題先由 `自我檢查.bat` 處理；本清單用來確認工程資料、版面與附件完整性。

目前適用工具版本：`V3.0.5`。

## 1. 工具狀態

- [ ] 使用 `開啟石材計算書.bat` 開啟工具。
- [ ] 狀態資訊顯示前端 HTML 與伺服器 HTML 均為 `石材計算書產生器_規範版V2.html`。
- [ ] 工具版本、伺服器版本與輸出檔名版本一致。
- [ ] 已執行 `自我檢查.bat`，且結果為 `PASS: all checks completed successfully`。

## 2. 專案基本資料

- [ ] 工程名稱、業主或施工廠商名稱已填寫。
- [ ] 工程地點、設計日期、檢核範圍已填寫。
- [ ] 技師姓名、執業登記號、複核人員已確認。
- [ ] 若正式送件需要簽章頁，已啟用簽章頁並確認顯示內容。
- [ ] 若需附技師執業證書，已上傳並確認頁面順序。

## 3. 設計條件

- [ ] 風壓來源已確認為 C&C、自訂風壓或採用高度表。
- [ ] 正負風壓方向性設定已確認。
- [ ] SDS、Ip、石材厚度、單位重、材料強度已確認。
- [ ] 若採非預設規範 profile，Rp、ap 與外牆 Ip 最小值已同步為該 profile 預設；人工覆寫值及採用理由已確認。
- [ ] 重新載入或匯入專案 JSON 後，規範參數總覽的 profile、Rp / ap / Ip 輸入與原專案一致，未沿用其他專案的瀏覽器狀態。
- [ ] 若舊版或外部專案缺少 profile-owned 欄位，已確認畫面提示與 `meta.applied_defaults` 所列補值欄位合理；專案明示的人工值仍保持不變。
- [ ] 插銷直徑與 Fy 已確認；V2 預設為 4 mm、Fy = 1350 kgf/cm2。
- [ ] 錨栓、背扣、插銷或角鋼規格與圖說一致。

## 4. 案例與檢核結果

- [ ] 至少建立一個有效案例。
- [ ] 每個案例的板寬、板高、固定形式、支承點數與實際配置一致。
- [ ] 未通過項目已完成工程判斷，必要時已修正尺寸或材料。
- [ ] 警示項目已確認，例如孔距、邊距、厚度、面積、層間變位或伸縮縫。
- [ ] 審查摘要頁的案例數、未通過項目與主要控制項目合理。

## 5. 附錄與追溯資料

- [ ] 規範條文對照表已依需求納入或關閉。
- [ ] 敏感度分析附錄已依需求納入或關閉。
- [ ] 範本來源與版本顯示正確。
- [ ] 附圖、照片、型錄、ETA/ACI 355.2 試驗報告等附件已依案件需求補齊。
- [ ] 匯出後同資料夾有產生 `_驗證報告.json` 稽核檔。
- [ ] 稽核檔內 `summary.dcr_method` 為 `normalized_by_check_sense`，且 `worst` 與 `worst_warning` 已分流。
- [ ] 稽核檔內 `review.server_evaluation.tool_html_match` 為 `true`。
- [ ] 稽核檔內 `review.server_status_not_evaluated` 為 `false`。
- [ ] 若已有前次輸出稽核檔，執行 `python audit_compare.py --latest --fail-on-regression` 且未出現退步。

## 6. 輸出檔案

- [ ] 檔名包含工程名稱、工具版本與時間戳。
- [ ] Word 或 PDF 開啟後，封面、目錄、審查摘要、主文、附錄順序正確。
- [ ] 表格沒有明顯截斷、重疊或跑版。
- [ ] 頁首頁尾與頁碼正常。
- [ ] 正式歸檔時，同步保存輸出檔、稽核 JSON、專案 JSON 與必要附件。

## 7. 修改工具後的額外步驟

- [ ] 若改過 V2 HTML、server、auto_word 或 generate_docx，重新執行 `自我檢查.bat`。
- [ ] 若改過計算公式或預設參數，更新 `js/regression-smoke.test.js` 的基準值並註記原因。
- [ ] 若改過內建範本，更新 `TEMPLATE_CATALOG_VERSION`。
- [ ] 若調整正式輸出版面，至少用一個標準插銷案例與一個背扣案例各匯出一次確認。
